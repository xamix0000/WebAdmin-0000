Архитектура взаимодействия узлов (SOA & Agentless)

Программный комплекс построен по сервисно-ориентированной архитектуре (SOA) с использованием безагентной модели. Ядро (Backend) реализовано на Python 3 (Flask) и выполняется под управлением промышленного WSGI-сервера Gunicorn на ОС Debian Linux.

1. Модульная структура (Flask Blueprints)

Для обеспечения масштабируемости (Scalability) монолитное приложение было разделено на изолированные модули Blueprints:

  • auth.py — Сквозная авторизация, работа с сессиями и проверка RBAC (ролей).
  • dashboard.py — Агрегация метрик, графов (Topology) и аналитики ИБ в реальном времени.
  • users.py / shares.py — Изолированные контроллеры для CRUD-операций (Создание, Чтение, Обновление, Удаление) над объектами инфраструктуры.
2. Решение проблемы "Эффекта наблюдателя"

Для предотвращения перегрузки серверов (Denial of Service) постоянными WMI-запросами со стороны интерфейса мониторинга, панель использует Многопоточный асинхронный поллинг (Multi-threaded Polling).

  • Background Workers: Изолированные потоки (Daemon Threads) собирают телеметрию каждые 1-5 секунд и записывают её в оперативную память (Глобальный словарь SYSTEM_CACHE).
  • Thread Safety (Потокобезопасность): Для предотвращения состояния гонки (Race Condition) и взаимных блокировок (Deadlocks) при обращениях к кэшу билетов Kerberos используется механизм семафоров threading.Lock().
  • Stateless API: Браузер клиента (JS) обращается только к кэшу бэкенда, получая данные со скоростью < 1 мс, что снижает нагрузку на ядро Windows Server до нуля.
3. Интеграция с Windows Server (WinRM)

Связь осуществляется через защищенный протокол WinRM. Для решения проблем с кодировками (кириллицей) используется инкапсуляция: скрипт PowerShell выполняется на сервере Windows, конвертирует ответ в JSON, кодирует в Base64 и возвращает в Python. Это обеспечивает 100% целостность данных без искажений CP866/UTF-8.

Навигация по модулям панели

Подробное описание функционала каждой вкладки управления системой WebAdmin.

🏠 Главная (Dashboard)

Единый центр мониторинга. Отображает ключевые метрики инфраструктуры (KPI), анимированные графики загрузки процессора и оперативной памяти узлов Debian и Windows Server. Содержит живой Журнал активности администратора и встроенный Web-Терминал для быстрого выполнения консольных команд (Bash/CMD/PS).

👥 Пользователи (Users Management)

Модуль управления учетными записями Active Directory. Позволяет создавать пользователей с применением строгих политик паролей (запрет смены, бессрочный пароль) и привязывать их к группам доступа. Через профиль пользователя доступен сброс пароля, принудительная блокировка и управление членством в группах.

🏢 Группы (Groups Management)

Модуль управления глобальными группами безопасности (Security Groups). Изолирует системные Builtin-группы для безопасности интерфейса. Позволяет создавать новые отделы, просматривать списки участников и массово исключать/добавлять сотрудников в проектные группы.

📁 Файлы / Samba (Share Management)

Оркестратор файловых ресурсов. Автоматизирует создание сетевых каталогов с жестким разделением прав (Только Чтение или Полный Доступ) для групп и индивидуальных пользователей. Поддерживает скрытие папок, фильтрацию по IP-адресам и имеет встроенный файловый навигатор с доступом к .Корзине.

🕸️ Граф сети (Topology Map)

Интерактивная карта активных сетевых соединений в реальном времени. Визуализирует, какие клиентские устройства (с указанием IP и логина) подключены к каким файловым ресурсам сервера. Указывает версию используемого протокола шифрования (например, SMB3_11).

🛡️ Аудит ИБ (Security Audit)

SOC-мониторинг уязвимостей. Анализирует базу Active Directory на предмет "мертвых душ", устаревших паролей, нарушений принципа Zero Trust (избыток администраторов) и попыток брутфорса. Доступен экспорт отчетов в CSV и PDF.

Информационная безопасность и криптография

1. Сквозная авторизация (SSO) и RBAC

Вход в веб-панель защищен делегированной аутентификацией. Python использует системный класс System.DirectoryServices.AccountManagement для валидации учетных данных напрямую в ядре AD. Доступ предоставляется исключительно пользователям, входящим в группы Domain Admins или Администраторы домена.

2. Защита канала управления (Kerberos)

Система полностью отказалась от устаревшего и уязвимого протокола NTLM в пользу криптографического стандарта Kerberos (RFC 4120). Процесс взаимодействия:

  1. Бэкенд запрашивает TGT (Ticket Granting Ticket) у KDC контроллера домена.
  2. Билет кэшируется в подсистеме Linux (/tmp/krb5cc_0).
  3. Связь с WinRM устанавливается исключительно по FQDN-имени с предъявлением сервисного билета (SPN: HTTP/dc.domain.com). Хеши паролей по сети не передаются.
3. Решение конфликтов кодировок (CLIXML/Base64)

Для предотвращения потерь кириллических данных и подавления системных ошибок PowerShell (Progress Bar CLIXML), все скрипты на стороне Windows инкапсулируют ответ в JSON, а затем конвертируют в Base64. Бэкенд получает чистую криптостроку, исключая любые искажения кодировок (CP866/UTF-8).

Файловая система и Изоляция (Samba)

Отказ от POSIX ACL в пользу динамических групп

Классическое управление сетевыми папками через setfacl или chmod 777 несет риски безопасности и вызывает сбои трансляции системных Builtin-групп AD. В данной панели применена архитектура изоляции на уровне ОС:

  • При создании ресурса генерируется изолированная локальная группа Linux (напр. smb_reports).
  • Директории назначаются жесткие права 2770 (SetGID). Доступ имеют только Root и локальная группа.
  • Разграничение прав делегировано демону Samba. Параметр valid users фильтрует доступ по спискам AD (Группы и Пользователи).
  • В случае успешной авторизации, Samba применяет параметр force group = smb_reports, маскируя доменного пользователя под локальную группу и предоставляя доступ к файлам без конфликта системных прав.
Система VFS Recycle (Анти-уничтожение)

Для защиты от случайного удаления данных (которое в Windows по сети производится безвозвратно), задействован модуль vfs_recycle. Файлы скрыто перемещаются в директорию .Корзина с сохранением структуры (keeptree) и поддержкой версионирования.

SOC Dashboard (SIEM Аналитика)

Выявление уязвимостей (Real-time Audit)

Модуль безопасности автоматически парсит метаданные учетных записей Active Directory и сопоставляет их с принципами Zero Trust. Анализатор выявляет:

  • Скомпрометированные учетки: атрибут LockedOut (попытки брутфорса).
  • Нарушения парольной политики: PasswordNotRequired и PasswordNeverExpires.
  • Мертвые души: отсутствие входов (LastLogonDate) более 90 дней.
  • Нарушения привилегий: превышение допустимого лимита администраторов домена.

Инциденты мгновенно отображаются в таблице алертов с возможностью выгрузки в CSV/PDF для формирования отчетов.

Журнал активности (SQLite)

Все действия операторов панели логируются во встроенную реляционную базу данных SQLite. Фиксируются: время, администратор, действие и целевой объект. Это обеспечивает неотрекаемость (Non-repudiation) вносимых изменений (в данной версии записываются последние 7 действий, в будущем возможно увеличение).

Руководство Администратора

Практическое руководство по управлению системой и решению нештатных ситуаций.

Используется нативный синтаксис Samba (параметр hosts allow). Примеры:
  • 192.168.56.0/24 - Разрешить доступ всей подсети (рекомендуется).
  • 192.168.56. (с точкой на конце) - Аналог подсети.
  • 192.168.56. EXCEPT 192.168.56.100 - Разрешить подсети, кроме конкретного узла.

Панель исключает конфликты прав на этапе создания.
• Если пользователь добавлен в ПОЛНЫЙ ДОСТУП, он получает права на запись и удаление файлов.
• Если пользователь добавлен в ТОЛЬКО ЧТЕНИЕ, он может просматривать и копировать файлы, но не изменять их.
Важно: Если пользователь состоит в группе, которой дано Чтение, но индивидуально ему выдана Запись — приоритет отдается Записи (расширение привилегий).

Модуль vfs_recycle перехватывает команды на удаление от Windows-клиентов и переносит файлы в скрытую директорию .Корзина внутри каждой шары с сохранением структуры каталогов (keeptree).

Если вкладка "Аудит ИБ" сигнализирует о заблокированной учетной записи (индикатор "Подбор пароля"):
  1. Перейдите на вкладку Пользователи.
  2. Найдите пользователя (у него будет серый статус "Заблокирован").
  3. В меню "Действия" выберите Разблокировать.
  4. Если пользователь забыл пароль, перейдите в его Профиль и выполните принудительный сброс пароля.

Терминал является Stateless (атомарным, без сохранения состояния сессии между командами). Для выполнения интерактивных команд или переходов по папкам объединяйте их в одну строку:
• В Linux (Bash): cd /etc/samba && ls -la
• В Windows (PowerShell): Set-Location C:\ ; Get-ChildItem
Ограничение: Команды, требующие интерактивного ввода (например, top или nano), будут автоматически убиты по таймауту через 15 секунд для предотвращения зависания панели.
